# 사용자 이야기

## 목적

사용자 이야기는 기능의 필요성을 사용자 관점에서 정리하기 위한 문서입니다.

기능명세는 이 문서를 바탕으로 더 구체화합니다.

## 작성 기준

사용자, 목적, 기대 효과가 한 번에 드러나야 합니다.

## 핵심 사용자 이야기

| ID | 사용자 | 원하는 것 | 기대 효과 | 상태 |
| --- | --- | --- | --- | --- |
| US-001 | 일반 사용자 | 익숙한 화면 흐름으로 서비스를 이용하고 싶다 | 학습 부담 없이 기능을 사용할 수 있다 | 진행 |
| US-002 | 운영자 | 기능 변경 전에 문서로 범위를 확인하고 싶다 | 구현 전 합의가 쉬워진다 | 진행 |
| US-003 | 개발자 | 화면, 데이터, API 영향 범위를 먼저 확인하고 싶다 | 작업 누락과 충돌을 줄일 수 있다 | 진행 |
| US-004 | Codex | 구현 전에 관련 문서를 읽고 싶다 | 문서 의도에 맞게 구현할 수 있다 | 진행 |

## 활용 방식

- 제품 요구사항과 연결합니다.
- 기능명세 문서의 출발점으로 사용합니다.
- 테스트 시나리오 작성의 기준으로 활용합니다.
