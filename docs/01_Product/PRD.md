# 제품 요구사항 문서

## 목적

이 문서는 현재 앱의 확장 방향과 우선 구현 범위를 정리하는 기준 문서다.

프로젝트는 새로 만드는 제품이 아니라, 이미 화면과 기능이 존재하는 앱을 동호회 운영 플랫폼으로 발전시키는 작업이다.

## 현재 제품 상태

현재 앱은 아래 특징을 가진다.

- `Home`, `Club`, `Record` 중심 Navigation 구조를 가진다.
- 라운드 생성, 기록 입력, 결과 확인과 같은 기록 흐름이 이미 존재한다.
- 회원 관리와 Club 단위 운영 개념이 일부 들어와 있다.
- 공통 UI, 공통 Navigation, 공통 로직 구조가 이미 형성되어 있다.

## 현재 요구사항

| ID | 요구사항 | 상태 |
| --- | --- | --- |
| PRD-001 | 기존 구조를 먼저 문서화한다. | 진행 |
| PRD-002 | Bottom Navigation 구조를 유지한다. | 적용 |
| PRD-003 | 회비 관리 기능은 기존 앱 확장 방식으로 설계한다. | 진행 |
| PRD-004 | 회비 관리는 회원 상태 관리와 자금 현황 관리를 함께 다룬다. | 진행 |
| PRD-005 | 개발은 문서 승인 이후에만 진행한다. | 적용 |

## 1차 제품 범위

현재 가장 먼저 정리해야 할 범위는 아래와 같다.

1. 앱 구조 문서화
2. 회비 운영 정책
3. 자금 현황 정책
4. 회비 관리 기능명세
5. 회비 관리 도입 위치 검토

## 비범위

아래 항목은 현재 단계의 우선 범위가 아니다.

- 복잡한 자동 정산
- 고급 통계
- 외부 결제 연동
- 대규모 Navigation 재편
- 기존 기록 흐름 재설계

## 완료 기준

- 현재 앱 구조를 문서만 보고 이해할 수 있다.
- 회비 관리의 운영 원칙이 정리되어 있다.
- 회비 관리 화면이 어디에 들어갈지 판단할 수 있다.
- 개발 전에 필요한 문서가 최소한으로 갖춰져 있다.
