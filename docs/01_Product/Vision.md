# 제품 방향 문서

## 제품 정체성

이 프로젝트는 단순한 골프 기록 앱이 아니다.

목표는 **골프 동호회 운영 플랫폼**이다.

슬로건은 다음과 같다.

> 골프의 모든 경험을 하나로.

## 현재 제품 방향

- 기존 기록 기능을 유지한다.
- 동호회 운영에 필요한 관리 기능을 단계적으로 확장한다.
- 기록, 운영, 관리가 하나의 흐름으로 이어지도록 설계한다.
- 기존 사용자가 낯설지 않게 점진적으로 확장한다.

## 현재 시점의 핵심 과제

1. 현재 앱 구조를 정확히 이해한다.
2. 기존 UI와 Navigation을 문서로 고정한다.
3. 회비 관리 기능을 운영 관점으로 설계한다.
4. 이후 회원관리, 공지관리, 라운드 운영으로 확장한다.

## 제품 판단 기준

- 동호회 운영에 실제 도움이 되는가
- 기존 사용자 흐름을 해치지 않는가
- 현재 `React Native`, `Expo`, `TypeScript`, `Supabase` 구조에 무리 없이 들어가는가
- 사람이 문서만 보고도 방향을 이해할 수 있는가
