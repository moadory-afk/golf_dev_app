# 회비 관리 릴리즈 체크리스트

## 목적

이 문서는 회비 관리 기능을 실제 배포하기 전에 확인해야 할 항목을 정리한다.

릴리즈 판단은 기능 구현 여부만 보지 않고, 기존 앱 구조와 운영 문서가 함께 맞는지 확인한다.

## 문서 확인

- `Fee Management.md`가 최신 상태다.
- `Fee Main.md`, `Fee Detail.md`, `Fee Policy.md`, `Treasury.md`, `Treasury Ledger.md`가 구현과 일치한다.
- `Fee Data.md`가 실제 데이터 구조와 일치한다.
- `Fee API.md`가 실제 호출 흐름과 일치한다.
- 테스트 기준 문서가 갱신되어 있다.

## 기능 확인

- 회비 관리 메뉴가 `Club` 내부에 있다.
- 회비 관리 메인 화면이 가장 최근 회차를 먼저 보여준다.
- 회원 목록에 이름, 상태, 금액, 최근 변경일이 표시된다.
- 상태 배지에서 상태 변경이 가능하다.
- 회원 상세 화면에서 예외 정보와 이력을 확인할 수 있다.
- 자금 현황 화면에서 현재 잔액, 이번 달 입금, 이번 달 지급, 최근 5건을 확인할 수 있다.
- 입금/지급 등록, 수정, 삭제가 정상 동작한다.

## 권한 확인

- 총무와 관리자만 회비 상태를 변경할 수 있다.
- 총무와 관리자만 입금/지급을 등록, 수정, 삭제할 수 있다.
- 일반 회원은 공개 범위 안에서만 조회할 수 있다.

## 구조 확인

- Bottom Navigation은 변경되지 않았다.
- 기존 `Home`, `Club`, `Record` 흐름이 유지된다.
- 회비 관리는 `Club` 내부 운영 기능으로 동작한다.
- 기존 화면 구조를 불필요하게 재편하지 않았다.

## 릴리즈 보류 기준

아래 항목 중 하나라도 해당하면 릴리즈를 보류한다.

- 기존 기록 기능에 영향이 있다.
- Bottom Navigation이 변경되었다.
- 문서와 구현이 다르다.
- 권한 기준이 제대로 적용되지 않았다.
- 자금 잔액 계산이 불명확하다.

## 최종 판단

아래 조건을 만족하면 릴리즈 가능으로 본다.

1. 기존 기능을 깨지 않았다.
2. 문서와 구현이 일치한다.
3. 총무가 실제 운영에 사용할 수 있다.
4. 일반 회원 공개 범위가 정책대로 적용된다.
5. 다음 확장 기능과 충돌하지 않는다.
