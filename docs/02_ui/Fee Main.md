# 회비 메인 화면 문서

## 목적

이 문서는 `Club` 안에서 진입하는 `회비 관리` 첫 화면의 정보 구조를 정리한다.

회비 관리는 하나의 화면으로 끝나는 기능이 아니라, 총무와 관리자가 실제 운영에 사용하는 화면 묶음으로 본다.

## 기본 구조

- 메뉴명은 `회비 관리`로 사용한다.
- 회비 관리 첫 화면은 2개 탭으로 구성한다.
- 탭 이름은 `회비관리 현황`, `회비납부 현황`으로 고정한다.
- 기본 진입 탭은 `회비관리 현황`으로 둔다.

## 화면 목표

- 총무가 들어오자마자 현재 자금 상황을 이해할 수 있어야 한다.
- 회원별 납부 상태는 별도 탭에서 빠르게 확인할 수 있어야 한다.
- 돈의 흐름과 사람의 상태를 한 화면에서 섞지 않고 분리해서 보여준다.
- 기존 `Club` 운영 화면과 같은 분위기와 밀도를 유지한다.

## 탭 구성

### 1. 회비관리 현황

이 탭은 동호회 자금 흐름을 보는 운영 탭이다.

주요 정보는 아래와 같다.

- 현재 잔액
- 이번 달 입금 합계
- 이번 달 지급 합계
- 최근 거래 내역
- 입금 등록 버튼
- 지급 등록 버튼

권장 구성 순서는 아래와 같다.

1. 상단 요약 카드
2. 최근 거래 목록
3. 빠른 등록 액션

### 2. 회비납부 현황

이 탭은 회원별 납부 상태를 보는 관리 탭이다.

주요 정보는 아래와 같다.

- 회원 이름
- 납부 상태
- 금액
- 최근 변경일
- 상태 필터
- 회원 검색

상태 값은 아래 3개만 사용한다.

- `완납`
- `일부납`
- `미납`

## 탭별 역할 분리 원칙

- `회비관리 현황`은 돈의 흐름을 본다.
- `회비납부 현황`은 사람의 상태를 본다.
- 두 탭의 요약 카드와 목록은 서로 목적이 겹치지 않게 구성한다.
- 자금 흐름 데이터와 회원 상태 데이터는 같은 화면에 섞어 배치하지 않는다.

## 주요 동작

- 탭 전환
- 입금 등록 화면 이동
- 지급 등록 화면 이동
- 회원별 납부 상태 조회
- 상태 필터
- 회원 검색
- 회원 상세 진입

## UI 원칙

- 기존 앱의 `turf` 디자인 방향을 유지한다.
- 운영 화면답게 카드 중심이되 과하게 장식하지 않는다.
- 숫자 요약은 한눈에 읽히게 배치한다.
- 목록은 스캔이 쉬운 밀도로 유지한다.
- Bottom Navigation은 변경하지 않는다.
