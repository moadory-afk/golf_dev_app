# 회비 화면 문서

## 목적

회비 관리는 하나의 단순 목록 화면이 아니라, 운영 화면 집합으로 본다.

이 문서는 회비 관리 화면이 어떤 구조로 들어가야 하는지 정리하는 초안 문서다.

## 화면 구성 원칙

- 기존 UI 흐름을 유지한다.
- Bottom Navigation은 변경하지 않는다.
- 회비 관리는 `Club` 흐름 안에서 접근하는 방향을 우선 검토한다.
- 총무와 관리자가 가장 자주 보는 운영 화면이라는 점을 반영한다.

## 필요한 화면

1. 회비 관리 메인 화면
2. 회원별 회비 상세 화면
3. 회비 정책 설정 화면
4. 자금 현황 대시보드
5. 입금/지급 내역 화면
6. 입금/지급 등록 화면

상세 기준 문서:

- `Fee Main.md`
- `Fee Detail.md`
- `Fee Policy.md`
- `Treasury.md`
- `Treasury Ledger.md`

## 회비 관리 메인 화면

- 월별 기준으로 본다.
- 상단에는 총 회비 금액, 완납 인원, 일부납 인원, 미납 인원을 표시한다.
- 회원 목록은 `일부납` 우선 정렬을 기본으로 둔다.
- 상태 필터와 검색을 지원한다.

## 자금 현황 화면

- 현재 잔액
- 이번 달 입금
- 이번 달 지급
- 최근 내역 5건

이 4개를 한 화면에서 먼저 보여주는 대시보드 구조를 기본으로 한다.

## 현재 검토 포인트

- 회비 관리 진입 위치를 `Club` 내부에서 어떻게 배치할지
- 회비 상태 관리와 자금 현황을 한 메뉴 안에서 어떻게 구분할지
- 운영 화면과 일반 회원 조회 화면을 어떻게 나눌지

## 현재 판단

현재 문서 기준의 1차 판단은 아래와 같다.

- 회비 관리는 `Club` 내부 운영 메뉴로 배치한다.
- 첫 화면은 회비 상태 관리 메인 화면으로 둔다.
- 자금 현황은 회비 관리 하위의 별도 운영 화면으로 분리한다.
