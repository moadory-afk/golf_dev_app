# 의사결정 기록

## 현재 확정된 방향

- 이 프로젝트는 신규 앱이 아니라 기존 골프 앱을 동호회 운영 플랫폼으로 확장한다.
- Bottom Navigation은 `Home`, `Club`, `Record` 구조를 유지한다.
- 운영 기능은 새로운 Tab을 만들지 않고 `Club` 안에서 확장한다.
- `Club` 화면은 일반 소개 화면이 아니라 운영 진입 화면으로 본다.
- 회비 관리는 `회비관리 현황`과 `회비납부 현황`을 분리해 본다.
- 회비 관리 프로토타입은 실제 데이터 연결 전에 화면 구조와 역할 분리를 먼저 검증한다.

## 최근 확정된 결정

- 상단 `설정` 버튼은 제거한다.
- `클럽 정보`와 `운영 설정` 진입은 운영 메뉴 구조 안에서 처리한다.
- 관리자/총무는 `Club` 화면에서 운영 메뉴를 본다.
- 일반 회원은 운영 메뉴보다 기록 중심 흐름이 우선이다.
- 회비 관리 첫 화면은 전체 자금 흐름을 먼저 보는 방향으로 설계한다.

## 남아 있는 의사결정

- `Club` 화면을 `관리 | 기록` 탭 구조로 나눌 최종 시점
- 일반 회원에게 회비 화면을 어디까지 노출할지
- 회비관리 현황의 기본 조회 기간
- 회비납부 현황에서 검색과 필터의 우선순위
