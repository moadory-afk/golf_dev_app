# 회비 관리 기능명세

## 목적

이 문서는 회비 관리 기능의 1차 범위와 운영 기준을 정리한다.

현재 단계에서는 구현 세부보다 운영 구조와 화면 역할을 먼저 고정한다.

## 기능 성격

회비 관리는 단순 납부 체크 기능이 아니다.

아래 두 축을 함께 다루는 운영 기능으로 정의한다.

1. 회원 회비 상태 관리
2. 동호회 자금 현황 관리

## 진입 위치

- 회비 관리는 `Club` 내부 운영 메뉴로 둔다.
- Bottom Navigation은 유지한다.
- `회원 관리`와 가까운 위치의 운영 메뉴로 배치한다.
- 관리자/총무가 `Club` 화면에서 바로 진입할 수 있어야 한다.

## 화면 구조

회비 관리 첫 화면은 아래 2개 탭으로 구성한다.

1. `회비관리 현황`
2. `회비납부 현황`

### 회비관리 현황

이 탭은 자금 흐름 중심 화면이다.

- 현재 잔액
- 입금 합계
- 지급 합계
- 최근 거래 내역
- 입금 등록
- 지급 등록

### 회비납부 현황

이 탭은 회원 상태 중심 화면이다.

- 회원 이름
- 상태
- 금액
- 최근 변경일
- 검색
- 상태 필터

## 납부 상태 기준

- 상태 값은 `완납`, `일부납`, `미납` 3개만 사용한다.
- 분납은 별도 방식으로 두지 않는다.
- 일부만 낸 경우는 `일부납`으로 처리한다.
- 상태 변경은 총무 또는 관리자만 수행한다.

## 회비 정책 기준

- 회비 방식은 동호회 단위로 운영한다.
- 월납과 연납을 지원한다.
- 회원별 예외 금액은 허용한다.
- 회비 회차는 총무 개입 없이 자동 생성하는 방향을 기본으로 한다.

## 자금 현황 기준

- 자금 현황은 회비 전용 장부가 아니라 동호회 운영 자금 흐름으로 본다.
- 입금과 지급은 별도 거래로 기록한다.
- 잔액은 전체 거래 누적 기준으로 계산한다.
- 거래 등록은 총무 또는 관리자만 가능하다.

## 권한 기준

- 관리자/총무: 조회 및 수정 가능
- 일반 회원: 조회 범위만 허용
- 일반 회원 공개 범위는 추후 운영 정책으로 세분화한다.

## 1차 범위

1. `Club` 운영 메뉴에서 회비 관리 진입
2. 회비 관리 2탭 화면
3. 회원별 납부 상태 목록
4. 자금 현황 요약 및 최근 거래
5. 입금/지급 등록 진입 구조

## 제외 범위

- 외부 결제 연동
- 고급 통계
- 자동 알림
- 복잡한 감사 이력
