# 회비 관리 테스트 기준

## 목적

이 문서는 회비 관리 기능을 개발하기 전, 무엇을 기준으로 정상 동작을 판단할지 정리한다.

개발 이후에는 이 문서를 기준으로 수동 확인과 자동 테스트 범위를 결정한다.

## 테스트 범위

회비 관리 테스트는 아래 두 축으로 나눈다.

1. 회원 회비 상태 관리
2. 자금 현황 관리

## 회원 회비 상태 관리

### 회비 정책

- 동호회는 `월납` 또는 `연납` 중 하나의 방식만 사용한다.
- 기본 금액이 정상 표시된다.
- 예외 금액은 특정 회원에게만 적용된다.
- 공통 마감일이 정상 반영된다.

### 회비 회차

- 월납은 새 달에 자동 생성된다.
- 연납은 총무가 필요할 때 수동 생성한다.
- 회비 관리 메인 화면은 가장 최근 생성된 회차를 먼저 보여준다.

### 상태 관리

- 상태는 `완납`, `미납`, `일부납`만 사용한다.
- 상태 배지를 누르면 상태 선택 메뉴가 열린다.
- 목록에서 바로 상태 변경이 가능하다.
- 회원 상세 화면에서도 상태 변경이 가능하다.
- 일괄 변경은 회원 선택 후 하단 액션바에서 처리한다.

### 권한

- 총무와 관리자만 상태를 변경할 수 있다.
- 일반 회원은 공개 범위 안에서 조회만 할 수 있다.

## 자금 현황 관리

### 대시보드

- 현재 잔액이 정상 표시된다.
- 이번 달 입금 합계가 정상 표시된다.
- 이번 달 지급 합계가 정상 표시된다.
- 최근 내역은 5건만 기본 표시된다.
- 입금/지급 등록 버튼이 노출된다.

### 입금/지급 내역

- 입금과 지급은 독립 내역으로 기록된다.
- 입금 등록 시 잔액이 증가한다.
- 지급 등록 시 잔액이 감소한다.
- 수정과 삭제가 정상 동작한다.
- `전체`, `입금`, `지급` 전환이 정상 동작한다.

## 구조 확인

- Bottom Navigation은 변경되지 않는다.
- 회비 관리는 `Club` 내부에서 진입한다.
- 기존 기록 흐름과 섞이지 않는다.
- `features` 폴더의 역할은 Business Logic으로 유지된다.

## 수동 확인 항목

- 총무가 첫 화면에서 현재 회비 현황을 바로 이해할 수 있는가
- 일반 회원 화면과 운영자 화면이 혼동되지 않는가
- 기존 앱의 색상, 카드, 헤더, 목록 스타일과 어울리는가
- 모바일 화면에서 상태 변경 동작이 부담스럽지 않은가
