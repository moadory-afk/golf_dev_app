# 회비 API 초안

## 목적

이 문서는 회비 관리 기능 구현 전에 필요한 API 동작을 정리하는 초안 문서다.

현재 단계에서는 실제 엔드포인트 이름보다 화면과 데이터가 어떤 동작을 필요로 하는지 정의한다.

## 기본 방향

API는 아래 세 영역으로 나눈다.

1. 회비 정책
2. 회원 회비 상태
3. 자금 현황

## 회비 정책 API

필요한 동작:

- 회비 정책 조회
- 회비 정책 생성
- 회비 정책 수정
- 공개 범위 설정 수정

사용 화면:

- 회비 정책 설정 화면

## 회원 회비 상태 API

필요한 동작:

- 가장 최근 회비 회차 조회
- 월별 회비 현황 조회
- 회원별 회비 상세 조회
- 회원 회비 상태 변경
- 회원 회비 상태 일괄 변경
- 특정 회원 제외 처리
- 예외 금액 적용

사용 화면:

- 회비 메인 화면
- 회원별 회비 상세 화면

## 자금 현황 API

필요한 동작:

- 자금 대시보드 조회
- 시작 잔액 조회
- 시작 잔액 설정
- 입금 목록 조회
- 지급 목록 조회
- 입금 등록
- 지급 등록
- 입금 수정
- 지급 수정
- 입금 삭제
- 지급 삭제

사용 화면:

- 자금 현황 화면
- 입금/지급 내역 화면
- 입금/지급 등록 화면

## 회비 메인 응답 기준

회비 메인 화면은 한 번의 조회로 아래 정보를 가져올 수 있어야 한다.

- 현재 조회 회차
- 총 회비 금액
- 완납 인원
- 일부납 인원
- 미납 인원
- 회원별 상태 목록

회원별 상태 목록에는 아래 정보가 필요하다.

- 회원 식별값
- 회원 이름
- 상태
- 적용 금액
- 최근 변경일

## 자금 대시보드 응답 기준

자금 현황 화면은 한 번의 조회로 아래 정보를 가져올 수 있어야 한다.

- 현재 잔액
- 이번 달 입금 합계
- 이번 달 지급 합계
- 최근 내역 5건

최근 내역은 입금과 지급을 섞어서 최신순으로 보여준다.

## 권한 기준

총무와 관리자:

- 회비 정책 조회/수정
- 회비 상태 변경
- 입금/지급 등록, 수정, 삭제

일반 회원:

- 공개 범위 안에서 회비 상태 조회
- 공개 범위 안에서 자금 현황 조회

## 오류 기준

필요한 기본 오류 상황:

- 권한 없음
- 동호회 없음
- 회비 정책 없음
- 회비 회차 없음
- 회원 없음
- 잘못된 상태값
- 잘못된 금액

## 1차 우선순위

1. 가장 최근 회비 회차 조회
2. 월별 회비 현황 조회
3. 회원 회비 상태 변경
4. 회원 회비 상태 일괄 변경
5. 자금 대시보드 조회
6. 입금/지급 목록 조회
7. 입금/지급 등록, 수정, 삭제

## 개발 전 검토 사항

- 현재 `Supabase` 접근 방식과 맞춰야 한다.
- 별도 서버 API가 필요한지, 클라이언트에서 `Supabase`를 직접 호출할지 결정해야 한다.
- 권한 정책은 데이터 구조 확정 후 함께 검토해야 한다.
