# 화면 이동 구조

## 현재 구조

현재 앱의 기본 Bottom Navigation은 아래 3개로 고정한다.

- `Home`
- `Club`
- `Record`

코드상 `History`로 표현된 영역은 사용자 관점에서 기록 흐름으로 이해한다.

## 고정 원칙

- 새 Bottom Tab은 추가하지 않는다.
- 기존 Tab의 역할을 임의로 바꾸지 않는다.
- Admin 성격의 운영 기능은 `Club` 안에서 우선 배치한다.
- 회비 관리도 새로운 독립 Tab이 아니라 기존 구조 안에서 해결한다.

## 현재 Stack 화면

현재 구조상 주요 상세 화면은 아래 흐름을 가진다.

- 프로필
- 설정
- 회원 관리
- 라운드 상세
- 스코어 캡처
- 스코어 확인 및 보정
- 라운드 결과
- 라운드 설정
- 스코어 입력

즉, 메인 흐름은 Bottom Navigation으로 유지하고, 상세 작업은 Stack 화면으로 확장하는 구조다.

## 회비 관리 도입 원칙

- 회비 관리는 `Club` 내부 운영 기능으로 우선 검토한다.
- 회비 정책, 회비 상태, 자금 현황은 같은 운영 묶음으로 배치한다.
- 기록 흐름과 운영 흐름을 섞지 않는다.
