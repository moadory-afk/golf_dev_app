# Club 구조 문서

## 목적

이 문서는 `Club` 영역 안에서 어떤 운영 기능을 배치하는지 정리하는 기준 문서다.

`Club`은 단순 소개 화면이 아니라 동호회 운영의 중심 화면으로 본다.

## Club 영역의 역할

`Club`은 아래 기능이 모이는 운영 영역이다.

- 클럽 기본 정보 확인
- 회원 관리
- 회비 관리
- 공지 관리
- 운영 설정
- 클럽 기록 확인

## 현재 구조 방향

현재 `Club` 화면은 아래 두 성격을 함께 가진다.

1. 운영 기능 진입
2. 클럽 기록 확인

초기 단계에서는 한 화면 안에 두 영역을 세로로 배치한다.

- 상단: 클럽 관리
- 하단: 클럽 기록

향후 데이터와 기능이 늘어나면 `관리 | 기록` 상단 탭 구조로 분리하는 방향을 검토한다.

## 권장 정보 구조

```text
Club
 ├── 클럽 관리
 │    ├── 클럽 정보
 │    ├── 회원 관리
 │    ├── 회비 관리
 │    ├── 공지 관리
 │    └── 운영 설정
 └── 클럽 기록
      ├── 핸디캡 랭킹
      ├── 명예의 전당
      └── 최근 클럽 라운드
```

## 회비 관리 위치 원칙

- 회비 관리는 `Club` 내부 운영 메뉴에 둔다.
- `회원 관리`와 가까운 위치에 둔다.
- `회비관리 현황`과 `회비납부 현황`은 같은 기능 묶음 안에서 탭으로 구분한다.
- 자금 현황은 회비 관리의 하위 성격으로 유지한다.

## 권한 기준

- 관리자/총무는 운영 메뉴를 본다.
- 일반 회원은 기록 영역 중심으로 이용한다.
- 같은 `Club` 화면 안에서도 권한에 따라 노출 정보가 달라질 수 있다.

## 설계 원칙

- 새로운 Bottom Tab은 추가하지 않는다.
- 기존 `Home`, `Club`, `Record` 구조를 유지한다.
- 운영 기능은 `Club` 안에서 확장한다.
- 화면 진입 방향이 중복되지 않도록 버튼 역할을 분명히 나눈다.
