# AI 작업 기준

이 문서는 이 프로젝트에서 AI가 어떤 역할을 맡는지 정리합니다.  
문서는 사람에게 먼저 도움이 되어야 하고, AI는 그 규칙을 따라야 합니다.

## 역할 분담

| 주체 | 역할 |
| --- | --- |
| ChatGPT | 구조 정리, 계획 정리, 문서 보강 |
| Codex | 구현, 수정, 테스트 반영 |
| GitHub | 변경 이력 관리, 협업 기록 |
| Obsidian | 프로젝트의 단일 기준 문서 |

## 작업 순서

1. [[START HERE|프로젝트 시작하기]]를 먼저 읽습니다.
2. [[PROJECT_CONSTITUTION|프로젝트 헌법]]으로 규칙을 확인합니다.
3. [[DASHBOARD|프로젝트 대시보드]]로 현재 상태를 봅니다.
4. 필요한 제품, 기능, UI, 데이터, API 문서를 읽습니다.
5. 문서에 맞춰 구현합니다.
6. 테스트와 릴리즈 체크리스트를 확인합니다.

## AI 책임

- 문서에 없는 내용을 임의로 만들지 않습니다.
- 기존 UI와 구조를 먼저 존중합니다.
- 기능을 바꾸기 전에 영향 범위를 확인합니다.
- 구현이 끝나면 문서도 함께 맞춥니다.

## Codex 책임

- 구현
- 수정
- 테스트 반영
- 문서에 적힌 흐름 유지

## ChatGPT 책임

- 제품 방향 정리
- 문서 초안 보강
- 의사결정 정리
- 용어 통일

## Obsidian 책임

- 문서의 기준점이 됩니다.
- 제품, 기능, 화면, 데이터, API를 서로 연결합니다.
- 오늘 작업한 내용을 다음 작업과 연결합니다.
