# 프로젝트 현황판

## 현재 진행 중

- `Club` 화면을 운영형 구조로 재정리
- 회비 관리 UI 프로토타입 정리
- 회비 관리 2탭 구조 확정
- 관리자/총무 기준 운영 흐름 문서화

## 다음 작업

- `회비 관리` 화면에 `회비관리 현황 | 회비납부 현황` 탭 구조 반영
- `Club` 화면의 `관리 | 기록` 분리 여부 최종 판단
- 회비 관리 화면의 실제 데이터 연결 범위 정리
- 총무/일반 회원 권한별 노출 기준 구체화

## 최근 완료

## 최근 변경사항

- 회비 관리는 단일 목록 화면이 아니라 운영 화면 집합으로 확정했다.
- 회비 관리의 기본 구조를 `회비관리 현황`과 `회비납부 현황` 2개 탭으로 정리했다.
- `Club` 화면은 관리자/총무가 운영 기능에 진입하는 중심 화면으로 사용한다.
- `회원 관리`, `회비 관리`, `공지 관리`, `운영 설정`을 운영 메뉴로 정리했다.
- 상단 `설정` 버튼은 역할 중복으로 제거하고, `클럽 정보` 및 `운영 설정` 진입으로 통합했다.
