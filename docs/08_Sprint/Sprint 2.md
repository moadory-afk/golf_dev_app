# Sprint 2

## 목표

회비 관리 기능을 개발 가능한 수준까지 문서와 화면 구조로 고정한다.

## 현재 상태

회비 관리 기능은 실제 데이터 연결 전 단계다.

현재는 아래 작업이 중심이다.

- 운영 정책 정리
- UI 프로토타입 검증
- 권한 구조 확인
- 화면 역할 분리

## 현재 확정 내용

- 회비 관리는 `Club` 운영 메뉴에서 진입한다.
- 회비 관리 첫 화면은 `회비관리 현황 | 회비납부 현황` 2탭 구조로 간다.
- `Club` 화면은 운영 메뉴와 기록 영역을 함께 가진다.
- 상단 `설정` 버튼은 제거한다.

## 1차 개발 범위

- `Club` 운영 메뉴 구성
- 회비 관리 프로토타입 화면
- 공지 관리 프로토타입 화면
- 회비 관리 2탭 정보 구조
- 관리자/총무 진입 흐름 확인

## 제외 범위

- 외부 결제 연동
- 자동 알림
- 고급 통계
- Navigation 구조 개편
