# 개발 승인 체크포인트

## 목적

이 문서는 회비 관리 개발을 시작하기 전에 반드시 확인해야 할 승인 기준을 정리한다.

사용자의 명시적 승인 전에는 개발을 시작하지 않는다.

## 현재 상태

- 회비 관리 정책: 초안 정리 완료
- 화면 구성: 초안 정리 완료
- 데이터 구조: 초안 정리 완료
- API 동작: 초안 정리 완료
- 테스트 기준: 초안 정리 완료
- 실제 개발: 1차 정적 UI 프로토타입 착수

## 개발 전 필수 확인

| 항목 | 상태 |
| --- | --- |
| `Club` 안에 회비 관리 진입 | 검토 필요 |
| 회비 메인 화면 구성 | 검토 필요 |
| 회원 상세 화면 구성 | 검토 필요 |
| 자금 현황 화면 구성 | 검토 필요 |
| 데이터베이스 구조 | 검토 필요 |
| 권한 기준 | 검토 필요 |
| 1차 구현 범위 | 정적 UI 프로토타입으로 시작 |

## 승인 문구

개발을 시작하려면 사용자가 명확하게 아래와 같이 승인해야 한다.

```text
회비 관리 1차 개발 승인
```

또는 이와 같은 의미의 명시적 승인 문장이 필요하다.

## 승인 전 금지

- `src` 수정 금지
- `app` 수정 금지
- `assets` 수정 금지
- `package.json` 수정 금지
- `supabase` 수정 금지
- 실제 화면 구현 금지
- 데이터베이스 마이그레이션 작성 금지

## 승인 후 첫 개발 후보

승인 후에도 한 번에 전체 기능을 개발하지 않는다.

가장 작은 시작점은 아래 중 하나로 선택한다.

1. 정적 UI 프로토타입
2. 데이터베이스 마이그레이션 초안
3. `Club` 내부 메뉴 진입점

현재는 `정적 UI 프로토타입`부터 진행한다.

## 1차 개발 시작 기록

- `Club` 내부에 `회비 관리` 진입 카드를 추가한다.
- 실제 저장, API, Supabase 연동은 하지 않는다.
- 더미 데이터 기반으로 회비 관리 화면의 정보 구조만 확인한다.
