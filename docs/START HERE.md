# 프로젝트 시작하기

이 문서는 이 프로젝트의 문서 출발점입니다.  
새 작업을 시작할 때는 항상 이 문서부터 읽고, 그다음 원칙과 현황을 확인합니다.

## 문서 작성 원칙

- 프로젝트 문서는 기능 개발보다 먼저 작성합니다.
- AI는 문서를 먼저 읽고, 그다음 구현합니다.
- 실제 구현은 문서에 적힌 흐름을 따라갑니다.
- 확인되지 않은 내용은 문서에 적지 않습니다.

## 오늘 작업에서 중요한 점

- 회비 관리는 총무가 정책을 유연하게 설정할 수 있어야 합니다.
- Bottom Navigation은 `Home`, `Club`, `Record`를 유지합니다.
- 회비 화면은 관리 현황과 납부 현황을 나눠서 봅니다.
- 입금과 지급은 회계 흐름의 시작점이므로 우선 개발 범위로 둡니다.
